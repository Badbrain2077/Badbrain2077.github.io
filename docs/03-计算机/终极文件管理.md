## 管理原则
### 分类：
原则：待办归档思维，图书馆分类思维
-   step1：**创建“000-收集箱”文件夹**，用于存放未分类文件，视为待办事项。
-   step2：**创建“001-处理中”文件夹**，用于存放从收集箱捞出来要用的文件。
-   step3：创建“100-学习”、“200-生活”等自己需要的、高度概括性的一级文件夹。
- 子集继续编号
- -   **文件夹各层级命名示例：**100-学习；110-数学；111-汤家凤网课。
-   **为什么写序号？**方便自动排序。
-   **子层级用0~10表示不够用？**11用A表示，12用B表示，以此类推。

### 文件命名：

日期_文件内容_版本号_标签

蜗牛波奇酱-孤独摇滚-20230224ver1

一般把工作看成是短期的，把学习看成是长期的，如果你的资料仅仅对你的工作有用，就放在工作里；如果你觉得你的资料对你的人生有用，就放在学习里。

很棒，我的思路和你类似，但文件夹不一样。  
C 系统和软件（~/Appdata; 还有根目录那两个） 临时文件通通不管。  
C:/usr/ 这里是开发环境和工具（jdk node.js naven gradle docker容器配置文件数据目录）  
D C:/uesrs/user/ 这下面的文档图片视频音乐桌面 都转移到D:/users/user/  
D:/users/OneDrive 是同步盘目录  
D:/users/localWork 是代码库
